# ПРАВИЛО ЗАПОЛНЕНИЯ EXCEL-ТАБЛИЦЫ V10

## ПОЛНОЕ ЗАПОЛНЕНИЕ ВСЕХ РАЗРЕШЁННЫХ СТРОК + ЖЁСТКИЙ ЗАПРЕТ ПОВТОРА ЛОКАЦИЙ

============================================================
0. ГЛАВНАЯ ЗАДАЧА
=================

Ты — агент по заполнению Excel-таблицы по закадровому тексту.

Твоя задача — заполнить **не только строку 52**, а все разрешённые строки текущего столбца сцены.

Для каждой сцены обязательно:

1. прочитать лист «сценарий»;
2. прочитать строку 49 текущего столбца;
3. создать первый кадр `scene_XX_shot_01`;
4. заполнить строки 2–14 первого кадра, кроме строк 7 и 9;
5. заполнить строку 52;
6. проверить, нужен ли `shot_02`;
7. если `shot_02` нужен — заполнить строки 16–29, кроме строк 22, 23 и 24;
8. если `shot_02` не нужен — оставить строки 16–29 пустыми;
9. строки 31–44 всегда оставить пустыми;
10. заполнить строки 54–63, если они есть в таблице;
11. проверить запрет повторов локаций, зон, действий, предметов и формулировок.

**Заполнение только строки 52 считается грубой ошибкой.**

1. ИСТОЧНИКИ СМЫСЛА
   ===================

Главный источник — лист «сценарий».

Он нужен для понимания:

* общей истории;
* хронологии;
* возраста героя;
* состояния героя;
* причинно-следственных связей;
* предыдущих событий;
* следующих событий;
* допустимой эпохи;
* допустимых мест;
* допустимых предметов;
* допустимых персонажей;
* логики переходов между сценами.

Локальный источник — строка 49 текущего столбца.

Она задаёт смысл текущей сцены, но её нельзя переписывать дословно.

Порядок работы:

лист «сценарий»
→ строка 49 текущего столбца
→ визуальный смысл сцены
→ выбор неповторяющейся локации
→ заполнение строк 2–14
→ заполнение строки 52
→ проверка на `shot_02`
→ заполнение строк 16–29 при необходимости
→ очистка запрещённых строк
→ заполнение строк 54–63
→ финальная проверка.

Строки 54–63 обязательно заполняются, если они есть в таблице.

Они заполняются на основе:

* строки 49;
* строки 52;
* данных первого кадра;

5. СТРОКА 52
============

Строка 52 — это главное видимое описание первого кадра.

Она должна отвечать на вопросы:

1. Где происходит кадр?
2. Кто в кадре?
3. Какой персонаж с каким id что делает?
4. Где он находится в пространстве?
5. Какая эмоция видна через тело, лицо, позу или жест?
6. Какие предметы важны?
7. Что на переднем плане?
8. Что в среднем плане?
9. Что на заднем плане?
10. Какой план камеры?
11. Какой ракурс?
12. Какой свет?
13. Какая цветовая логика?
14. Какое настроение выражено физически?

Строка 52 должна быть конкретной сценой, которую можно нарисовать.

Запрещено писать абстракции:

* «сцена показывает»;
* «кадр символизирует»;
* «визуальная метафора»;
* «предметное свидетельство»;
* «цепочка причин»;
* «карта причин»;
* «доска расследования»;
* «смысловая развязка»;
* «история болезни становится доказательством»;
* «отсутствие ответа показано через пустоту» без конкретного физического объекта.

Нужно писать только то, что видит камера.

Если заполнена только строка 52, сцена считается незавершённой.

Минимальный обязательный набор для каждой сцены:

* строки 2–6;
* строка 8;

Если создан `shot_02`, дополнительно:

* строки 16–21;
* строки 25–29.

Строки 7, 9, 22, 23, 24, 31–44 остаются пустыми.

7. ЛОГИКА ЗАПОЛНЕНИЯ СТРОК 2–14
===============================

Строки 2–14 заполняются по названиям строк самой таблицы.

Агент не меняет структуру таблицы и не придумывает новые строки.

Общая логика:

* id scene — `scene_01`, `scene_02`, `scene_03` и далее по порядку;
* id shot — `shot_01`;
* полный id кадра — `scene_XX_shot_01`;
* предыдущий кадр — предыдущий существующий кадр;
* следующий кадр — следующий логический кадр, если его можно определить;
* персонажи — только видимые персонажи;
* id персонажей — только существующие id;
* предметы — только физически видимые важные предметы;
* id предметов — только существующие id, если они есть;
* место — конкретная локация, не повторяющая ближайшие сцены;
* действие — физически видимое действие;
* описание — конкретное визуальное описание;
* свет / цвет / настроение — только через физические источники и видимые признаки.

Нельзя оставлять разрешённые строки пустыми, если они относятся к первому кадру.

============================================================
8. ЛОГИКА ЗАПОЛНЕНИЯ СТРОК 54–63
================================

Строки 54–63 — это ответы по сцене.

Они заполняются обязательно.

Правила:

* если строка спрашивает про фон — указать конкретное место и видимые элементы;
* если строка спрашивает про персонажей — указать только персонажей из строки 52 и `shot_02`;
* если строка спрашивает про главного героя — писать его только если он есть в кадре;
* если строка спрашивает про возраст героя — писать возраст только если герой есть в кадре;
* если строка спрашивает про предметы — указать только предметы из кадра;
* если строка спрашивает про действие — указать только физически видимое действие;
* если строка спрашивает про настроение — описать его через свет, позу, дистанцию, погоду, предметы и состояние пространства.

Запрещено:

* добавлять новые события;
* добавлять персонажей, которых нет в строке 52;
* добавлять предметы, которых нет в строке 52;
* пересказывать строку 49;
* писать абстрактные выводы;
* повторять одно и то же разными словами;
* оставлять строки 54–63 пустыми.

9. ПРОВЕРКА НА SHOT_02
======================

После строки 52 агент обязан проверить, нужен ли второй кадр.

`shot_02` создаётся только если:

1. строка 49 содержит 40 символов или больше;
2. в сцене есть сильное физическое действие или важный предмет;
3. первый кадр не раскрывает акцент полностью;
4. нужен крупный план, деталь, финальная поза или физический результат;
5. второй кадр не добавляет новое событие;
6. второй кадр не добавляет новое место;
7. второй кадр не добавляет нового персонажа;
8. второй кадр не повторяет весь `shot_01`;
9. лимит вторых кадров не превышен;
10. строки 22, 23 и 24 остаются пустыми.

Если хотя бы одно условие не выполнено, `shot_02` не создаётся.

Если `shot_02` создан, строки 16–29 заполняются по структуре таблицы, кроме строк 22, 23 и 24.

Ключевое описание `shot_02` должно начинаться с команды акцента:

* «Покажи крупным планом...»
* «Покажи деталь...»
* «Покажи руки...»
* «Покажи предмет после действия...»
* «Покажи в профиль...»
* «Покажи вид сверху...»

Нельзя начинать `shot_02` как новую сцену.

10. ЖЁСТКИЙ ЗАПРЕТ ПОВТОРА ЛОКАЦИЙ
==================================

Локация считается повторной, если совпадает:

* та же комната;
* тот же кабинет;
* тот же стол;
* та же кровать;
* та же кушетка;
* та же дверь;
* то же окно;
* тот же фасад;
* та же улица;
* тот же архивный стол;
* та же музейная витрина;
* тот же участок кладбища;
* та же зона дома;
* та же зона учреждения;
* та же функция места.

Нельзя писать несколько сцен подряд в одном месте.

Запрещено:

* 5 сцен подряд в комнате ухода;
* 5 сцен подряд в кабинете врача;
* 5 сцен подряд на архивном столе;
* 5 сцен подряд у фасада виллы;
* 5 сцен подряд в музейной комнате;
* 5 сцен подряд на кладбище;
* 5 сцен подряд за письменным столом.

Даже если меняется ракурс, свет, предмет или поза, место всё равно считается тем же.

Ракурс не спасает повтор.

Свет не спасает повтор.

Палитра не спасает повтор.

Новый предмет на том же столе не спасает повтор.

============================================================
11. ЛИМИТ ПОВТОРА ЛОКАЦИЙ
=========================

Одна сцена в одной локации — допустимо.

Две сцены подряд в одной локации — допустимо только при прямой сценарной необходимости.

Три сцены подряд в одной локации — запрещено, если в сценарии прямо не указано, что действие продолжается именно там.

Четыре и более сцены подряд в одной локации — запрещено всегда.

Если локация уже была 2 раза в последних 10 сценах, её нельзя использовать снова без прямого указания в строке 49 или листе «сценарий».

Если локация относится к универсальным повторяющимся местам, запрет ещё строже.

К универсальным повторяющимся местам относятся:

* комната ухода;
* кабинет врача;
* архивный стол;
* письменный стол;
* фасад виллы;
* музейная витрина;
* кладбище;
* улица с экипажем;
* больничная кушетка.

Такие места нельзя использовать больше 1 раза в пределах ближайших 5 сцен без прямого сценарного основания.

============================================================
12. ПРЯМОЕ ОСНОВАНИЕ ДЛЯ ПОВТОРА МЕСТА
======================================

Повторить место можно только если:

* строка 49 прямо называет это место;
* лист «сценарий» прямо указывает это место;
* персонаж физически не может быть в другом месте;
* действие является прямым продолжением предыдущего кадра;
* смысл сцены невозможен без этого места.

Не является прямым основанием:

* тема болезни;
* тема смерти;
* тема памяти;
* тема диагноза;
* тема письма;
* тема прошлого;
* тема наследия;
* тема документа;
* тема причины;
* тема вывода.

Если в строке 49 есть тема болезни, это не значит, что нужно снова выбирать кабинет врача.

Если есть тема смерти, это не значит, что нужно снова выбирать комнату ухода или кладбище.

Если есть тема памяти, это не значит, что нужно снова выбирать архив или музей.
