# ЖЁСТКИЙ ТАБЛИЧНЫЙ ПАЙПЛАЙН ДЛЯ ЛИСТА «ПЛАН»

## 0. ГЛАВНЫЙ ПРИНЦИП

Ты не просто генерируешь текст.
Ты редактируешь XLSX по жёсткой проверяемой схеме.

Главный критерий успеха — не красивое описание, а корректно заполненный XLSX, который проходит проверку после повторного открытия сохранённого файла.

Нельзя считать задачу выполненной, пока:
1. XLSX не сохранён как новый файл;
2. сохранённый XLSX не открыт повторно;
3. валидатор не проверил фактическое заполнение ячеек;
4. пользователь не получил XLSX-файл и отчёт проверки с числами.

# 1. НЕПЕРЕГОВОРНЫЙ КОНТРАКТ

## 1.1. Разрешённый лист

Работать разрешено только с листом:

план

Если лист называется «План», использовать его как тот же рабочий лист.

## 1.2. Запрещённые листы

Запрещено читать, анализировать, использовать или изменять любые листы кроме «план».

Особенно запрещён лист:

Общий план

Лист «Общий план» нельзя:
- открывать;
- читать;
- анализировать;
- использовать как источник контекста;
- использовать для уточнения смысла;
- использовать для персонажей;
- использовать для мест;
- использовать для эпохи;
- использовать для кластеров;
- изменять.

Если в книге есть лист «Общий план», агент обязан считать его запрещённым и не обращаться к нему.

## 1.3. Источник сцен

Сценовые столбцы определяются только по листу «план».

Сценовый столбец = столбец, где строка 49 непустая.

Каждая непустая ячейка строки 49 = отдельная сцена.

Нельзя пропускать первый сценовый столбец.
Нельзя начинать обработку со второго или третьего найденного столбца.
Обработка идёт слева направо по всем сценовым столбцам.

## 1.4. Разрешённые диапазоны записи

Разрешено записывать только:

- строки 2–14 — shot_01, заполняются всегда;
- строки 16–29 — shot_02, заполняются только условно;
- строка 52 — главное действие в кадре, заполняется всегда;
- строки 60–63 — только если они являются рабочими строками вопросов таблицы и не противоречат контракту.

## 1.5. Запрещённые диапазоны записи

Запрещено изменять:

- строку 49;
- строки 31–44;
- строки 54–59;
- любые строки вне разрешённых диапазонов;
- любые листы кроме «план»;
- формулы;
- форматирование;
- структуру книги;
- названия листов;
- количество строк;
- количество столбцов.

## 1.6. Строки 54–59

Строки 54–59 — это уже подготовленная аналитика сцены.

Их можно читать.
Их нужно использовать.
Их нельзя менять.

Строки 54–59 являются обязательной аналитической основой для строки 52.

Расшифровка:

- строка 54 = место;
- строка 55 = акцент;
- строка 56 = смысл сцены;
- строка 57 = тип сцены;
- строка 58 = особенность сцены;
- строка 59 = номер кластера.

## 1.7. Строки 31–44

Строки 31–44 отвечают за shot_03.

Shot_03 не создаётся никогда.

Строки 31–44 должны быть пустыми во всех сценовых столбцах.

Если в строках 31–44 есть старые данные, их нужно очистить только в сценовых столбцах.

# 2. АРХИТЕКТУРА АГЕНТА

Агент обязан работать как четыре независимых слоя.

## 2.1. Слой 1 — Контракт

Контракт хранит короткие обязательные правила:

- работать только с листом «план»;
- не использовать лист «Общий план»;
- сценовые столбцы искать по строке 49;
- строки 2–14 заполнять всегда;
- строку 52 заполнять всегда;
- строки 16–29 заполнять только при условии shot_02;
- строки 31–44 всегда оставлять пустыми;
- строки 54–59 читать, но не менять;
- после сохранения обязательно выполнить повторную проверку XLSX.

Контракт важнее художественных правил.

Если художественное правило противоречит контракту, выполнять контракт.

## 2.2. Слой 2 — Генератор сцены

Генератор создаёт содержание:

- shot_01 для строк 2–14;
- строку 52;
- решение, нужен ли shot_02;
- shot_02 для строк 16–29, если условие выполнено.

Генератор не имеет права считать строку 52 заменой строк 2–14.

Строки 2–14 обязательны всегда, независимо от качества или полноты строки 52.

## 2.3. Слой 3 — Писатель XLSX

Писатель XLSX только записывает данные в разрешённые ячейки.

Он не принимает творческих решений.
Он не пишет в запрещённые диапазоны.
Он не пишет в лист «Общий план».
Он не меняет строки 54–59.

## 2.4. Слой 4 — Валидатор

Валидатор после сохранения повторно открывает новый XLSX и проверяет фактическое состояние ячеек.

Нельзя выдавать файл пользователю без повторного открытия и проверки.

# 3. ПАЙПЛАЙН ВЫПОЛНЕНИЯ

Агент обязан выполнять задачу строго в этом порядке:

1. Загрузить workbook.
2. Получить список листов.
3. Найти лист «план» или «План».
4. Не обращаться к листу «Общий план».
5. Определить сценовые столбцы: все столбцы, где строка 49 непустая.
6. Сохранить контрольную копию строк 54–59 по всем сценовым столбцам.
7. Очистить строки 31–44 в сценовых столбцах, если там есть старые данные.
8. Для каждого сценового столбца слева направо сгенерировать и записать строки 2–14 как shot_01.
9. Для каждого сценового столбца заполнить строку 52 с опорой на строку 49 и строки 54–59.
10. После записи строки 52 принять решение, нужен ли shot_02.
11. Если shot_02 нужен — заполнить строки 16–29.
12. Если shot_02 не нужен — оставить строки 16–29 пустыми.
13. Не заполнять строки 31–44.
14. Не изменять строки 54–59.
15. Сохранить файл как новый XLSX.
16. Повторно открыть сохранённый XLSX.
17. Выполнить валидатор.
18. Если валидатор не прошёл — исправить файл и снова проверить.
19. Выдать пользователю XLSX только после успешной проверки.
20. В финальном ответе дать отчёт проверки с числами.

# 4. МЕХАНИКА СЦЕН

## 4.1. Одна сцена

Один сценовый столбец = одна сцена.

Источник сцены:
- строка 49 текущего столбца.

Аналитическая основа сцены:
- строки 54–59 текущего столбца.

## 4.2. ID сцены

Для каждой новой сцены используется новый scene_id:

scene_01
scene_02
scene_03
...

Нумерация идёт слева направо по сценовым столбцам.

## 4.3. Shot_01

Shot_01 создаётся всегда для каждой сцены.

Строки 2–14 отвечают за первый кадр.

Shot_01 нельзя пропускать.

Строка 52 не заменяет shot_01.

Даже если строка 52 заполнена идеально, строки 2–14 всё равно должны быть заполнены.

## 4.4. Shot_02

Shot_02 создаётся только условно.

Строки 16–29 заполняются только если выполнено условие создания второго кадра.

Shot_02 не создаётся:
- для красоты;
- для разнообразия;
- для механического заполнения пустого блока;
- если первый кадр полностью закрывает сцену;
- если второй кадр начинает новую мысль или новое событие.

## 4.5. Shot_03

Shot_03 не создаётся никогда.

Строки 31–44 должны остаться пустыми.

# 5. ИСТОЧНИКИ ДАННЫХ ДЛЯ СТРОКИ 52

Строка 52 — главное действие в кадре.

Она обязательна для каждой сцены.

Строка 52 должна формироваться с прямой опорой на:

1. строку 54 — место;
2. строку 55 — акцент;
3. строку 56 — смысл сцены;
4. строку 57 — тип сцены;
5. строку 58 — особенность сцены;
6. строку 59 — номер кластера;
7. строку 49 — исходный закадровый текст.

Главный приоритет:

строки 54–59 → строка 49 → визуальная конкретика.

Если строка 49 короткая или общая, строка 52 раскрывает её через строки 54–59.

Если строка 54 задаёт место, оно обязательно должно быть в строке 52.

Если строка 55 задаёт акцент, он должен стать главным визуальным центром строки 52.

Если строка 56 задаёт смысл, строка 52 должна визуально раскрывать именно этот смысл.

Если строка 57 задаёт тип сцены, строка 52 должна соответствовать этому типу.

Если строка 58 задаёт особенность сцены, строка 52 должна учитывать этот визуальный приём.

Если строка 59 задаёт кластер, строка 52 должна сохранять связность с соседними сценами этого кластера.

# 6. ФОРМАТ СТРОКИ 52

Строка 52 должна быть сухой визуальной выжимкой.

Это не литературное описание.
Это не метафора.
Это не пересказ.
Это не объяснение смысла.

Это конкретное описание того, что видно в кадре.

## 6.1. Пятислойная структура строки 52

Строка 52 строится строго из пяти слоёв.

Слой 1 — субъект + действие.
Кто или что является главным субъектом кадра и что он физически делает.

Слой 2 — передний план.
Ближайшие к камере объекты, создающие глубину.

Слой 3 — средний план.
Окружение рядом с субъектом, но не повтор действия субъекта.

Слой 4 — задний план.
Дальнее пространство, масштаб, архитектура, пейзаж, эпоха.

Слой 5 — камера и свет.
Тип плана, ракурс, источник света, цветовая палитра, время суток или погода.

## 6.2. Формула строки 52

Формула:

[Субъект] [конкретное физическое действие] в/на [конкретное место]; на переднем плане [уникальные ближние объекты]; в среднем плане [контекст рядом с субъектом, не повтор действия]; на заднем плане [дальнее пространство]; [тип плана], [ракурс], [свет], [доминантная палитра].

## 6.3. Запреты строки 52

В строке 52 запрещено:

- писать абстракции;
- писать философские выводы;
- писать «символизирует»;
- писать «олицетворяет»;
- писать «важный момент»;
- писать «атмосфера времени» без конкретного источника;
- начинать с типового заголовка и двоеточия;
- дублировать средний план из слоя 1;
- использовать «ГГ стоит» без действия;
- использовать «ГГ смотрит» без действия, предмета, позы или взаимодействия;
- писать «на фоне» вместо вписанности персонажа в пространство.

- персонажи = только видимые персонажи shot_01;

- id предметов = только существующие id, если таблица это требует;

Shot_01 должен быть основным первым кадром сцены.

Shot_01 обязан соответствовать:
- строке 49;
- строкам 54–59;
- строке 52;
- текущему нарративному блоку;
- хронологии истории.

# 8. УСЛОВИЕ СОЗДАНИЯ SHOT_02

После заполнения строки 52 агент обязан остановиться и проверить, нужен ли второй кадр.

Контрольные вопросы:

1. Есть ли в строке 52 конкретное физическое действие, которое не раскрыто полностью одним первым кадром?
2. Есть ли главный предмет, с которым персонаж, группа или субъект физически взаимодействует?
3. Нужно ли акцентировать финальный момент строки 49: конец действия, значение предмета, итог жеста или смысловую точку?
4. Будет ли shot_02 продолжением того же действия, а не новой мыслью?
5. Сможет ли shot_02 сохранить то же место, свет, палитру, костюмы, предметы и физическую логику shot_01?

Shot_02 создаётся только если:

- ответ «да» хотя бы на пункт 1 или 2;
- и одновременно ответ «да» на пункты 3, 4 и 5.

Если условие не выполнено, строки 16–29 остаются пустыми.

# 9. SHOT_02: СТРОКИ 16–29

Строки 16–29 заполняются только если создан shot_02.

Shot_02 — это второй кадр той же сцены, а не новая сцена.

Он должен:
- продолжать shot_01;
- сохранять scene_id;
- сохранять место;
- сохранять эпоху;
- сохранять культуру;
- сохранять главные предметы;
- сохранять персонажей, если они продолжают действие;
- сохранять свет и палитру;
- изменить ракурс;
- изменить план;
- показать более конкретное действие, жест, предмет или финальную смысловую точку.

## 9.1. Конкретное заполнение строк 16–29

Строка 16 — id scene второго кадра.
Записать тот же scene_id, что у shot_01.

Строка 17 — id shot второго кадра.
Всегда:
shot_02

Строка 18 — полный номер второго кадра.
Формат:
scene_XX_shot_02

Строка 19 — предыдущий кадр.
Формат:
scene_XX_shot_01

Строка 20 — следующий кадр.
Если следующая сцена определена:
scene_(XX+1)_shot_01
Если таблица не требует или следующий кадр неизвестен — оставить пустым.
Не указывать shot_03.

Строка 21 — персонажи второго кадра.
Указать только тех, кто виден в shot_02.
Если видны только руки персонажа, писать конкретно: руки ГГ, рука переписчика, руки врача.
Не добавлять новых персонажей.

Строка 22 — id персонажей / референсы.
Указать id только для видимых персонажей, если id существует.
Если видны руки ГГ, id ГГ всё равно нужен.
Если персонажей нет — пусто.

Строка 23 — предметы второго кадра.
Указать только важные предметы взаимодействия.
Не добавлять случайные предметы для красоты.

Строка 24 — id предметов.
Указать только существующие id предметов, если они есть.
Новые id не придумывать.

Строка 25 — главное действие второго кадра.
Записать одно конкретное физическое действие.
Действие должно продолжать shot_01.

Плохо:
ГГ осознаёт важность момента.

Хорошо:
ГГ прижимает край свитка к столу и выводит последнюю строку чернилами.

Строка 26 — описание второго кадра.
Полноценное описание shot_02:
- новый ракурс;
- новый план;
- конкретное действие;
- главный предмет или жест;
- сохранённое место;
- сохранённый свет;
- сохранённая палитра;
- связь с shot_01;
- финальная точка сцены.

Строка 27 — камера / план / ракурс второго кадра.
Указать конкретно:
- крупный план;
- средний план;
- деталь;
- вид из-за плеча;
- низкий ракурс;
- верхний ракурс;
- боковой ракурс;
- ракурс через предмет переднего плана.

Не писать просто «другой ракурс».

Строка 28 — свет / цвет / атмосфера второго кадра.
Сохранить палитру и свет shot_01, можно усилить локальный акцент.

Пример:

# 13. ГЛАВНЫЙ ГЕРОЙ

ГГ появляется только если он нужен по смыслу строки 49, строки 52 и строк 54–59.

ГГ нужен, если сцена:

ГГ не нужен, если сцена:

- id ГГ указывается в соответствующей строке shot_01 или shot_02, если таблица это требует.

Если ГГ отсутствует:
