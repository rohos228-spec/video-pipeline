# Universal Script Prompt Blueprint

Этот файл нужен для создания новых промтов шага `02_script` по единой структуре.

Активный Blocks v2 шаблон находится здесь:

```text
prompts/steps/02_script/template.md
```

## Каноническая структура

Любой новый промт для закадрового текста должен раскладываться на 7 разделов:

1. `ТЕХНИЧЕСКАЯ ЧАСТЬ`
2. `РОЛЬ И ЗАДАЧА`
3. `ТЕМА`
4. `ИСТОЧНИКИ И РЕЖИМ`
5. `НАВЫКИ И СТРУКТУРА`
6. `ГОЛОС И АНТИ-GPT ФИЛЬТР`
7. `ФОРМАТ ВЫВОДА И САМОПРОВЕРКА`

Техническая часть всегда остаётся первым статическим блоком. Она описывает протокол шага, а не творческий стиль.

## Категории блоков

| Категория | Дефолтный блок | Назначение |
|---|---|---|
| `script_role` | `voiceover_author` | Роль сценариста и задача voiceover |
| `source_policy` | `xlsx_general_plan_only` | Что читать, откуда брать факты, что нельзя выдумывать |
| `script_mode_selector` | `universal_modes` | Автовыбор режима: герой, тема, процесс, предмет, история, наука |
| `script_domain_skills` | `biography_history_science_process_object` | Уникальные навыки по типам материала |
| `script_narrative_structure` | `short_voiceover_arc` | Хук, завязка, развитие, инсайт, финал |
| `script_continuity_rules` | `smooth_voiceover_flow` | Связность, ритм, текст для уха |
| `script_voice_tone` | `human_documentary_voice` | Человеческий документальный голос |
| `script_anti_gpt_patterns` | `zinser_filter` | Фильтр против машинных паттернов |
| `script_output_contract` | `voiceover_txt_60s` | `voiceover.txt`, лимиты, запрет служебного текста |
| `script_self_check` | `voiceover_quality_gate` | Финальная внутренняя проверка |
| `script_segmentation_rules` | `long_cells_110_140` | Опциональный long-form режим с ячейками |

## Как создать новый вариант

1. Не копируй весь старый промт в один файл.
2. Определи, какой навык меняется: роль, режим, структура, стиль, анти-GPT фильтр или формат вывода.
3. Создай новый `.md` в соответствующей категории `prompts/blocks/<category>/`.
4. Подключи его через Studio UI или через `prompt_overrides.blocks`.
5. Не меняй техническую часть, если не изменился реальный I/O шага.

## Long-form вариант

Для длинных сценариев не надо ломать 60-секундный контракт. Используй отдельный output-блок и добавляй:

```md
{{BLOCK:script_segmentation_rules}}
```

Long-form логика:

1. сначала цельный рассказ;
2. затем редактура;
3. затем деление на ячейки 110–140 символов;
4. финальная проверка швов между ячейками.
