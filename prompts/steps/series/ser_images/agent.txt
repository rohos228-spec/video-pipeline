ПРИМЕЧАНИЕ: Этот агент — не GPT-агент, создающий текст. Это операционный
runbook для автоматического пайплайна (Python/bash скрипт) или для оператора,
который запускает генерацию картинок в Outsee и обновляет статусы в Excel.
Если ты — GPT, читающий этот файл для контекста, смотри §3 «Источники» и §8
«Статус R66» — они описывают, что должно появиться после работы этого шага.

§1. РОЛЬ И НАЗНАЧЕНИЕ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Этот шаг отвечает за:
  1. Считывание промтов картинок (R45/R46/R47) из листа «план»
  2. Отправку каждого промта в Outsee (или другой AI image generator)
  3. Скачивание готовых изображений и сохранение в папку серии
  4. Обновление статуса R66 в «план»: «картинки_ок» или «переделать»
  5. Сигнал для человека о готовности галереи к просмотру

§2. МЕСТО В ПАЙПЛАЙНЕ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ПРЕДЫДУЩИЙ АГЕНТ: ser_12_image_prompts (GPT)
  → Заполнил R45/R46/R47 для всех утверждённых ракурсов.
  → R66 на этих ячейках: пусто или «ждёт_генерации».

ЭТОТ ШАГ:
  → Читает R45/R46/R47.
  → Генерирует изображения.
  → Сохраняет файлы.
  → Обновляет R66.

СЛЕДУЮЩИЙ АГЕНТ: ser_13_anim_prompts (GPT)
  → Стартует ТОЛЬКО после человеческого одобрения галереи.
  → Читает R66 = «картинки_ок» как сигнал.

ЧЕЛОВЕЧЕСКИЙ СТОП-КРАН:
  После обновления R66 пайплайн отправляет уведомление человеку.
  Человек просматривает галерею. Принятые кадры: R66 остаётся «картинки_ок».
  Отклонённые: R66 = «переделать» (возможно с новым промтом от ser_12).
  Только после одобрения → запуск ser_13_anim_prompts.

§3. ВХОДНЫЕ ДАННЫЕ (ЧТО ЧИТАЕМ)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Файл: project.xlsx (серия)
  Лист: «план»
  Строки: R45 (промт кадр1), R46 (промт кадр2), R47 (промт кадр3)
  Фильтрация: только ячейки, где:
    - ячейка непустая
    - R66 для данного кадра ≠ «картинки_ок» (не перегенерировать одобренные)
    - R67 = пусто ИЛИ ≠ «да» (не трогать заблокированные после picture lock)

  Метаданные из «план» (для именования файлов):
    - id scene, id shot (из блока кадра)
    - номер кадра (для формирования имени файла)

§4. ПОРЯДОК ГЕНЕРАЦИИ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ШАГ 1. Открыть Excel, прочитать «план», собрать очередь:
  Список (промт, id сцены, id shot, номер ячейки R45/R46/R47)
  для всех непустых непроверенных ячеек.

ШАГ 2. Для каждого промта в очереди:
  2.1. Отправить промт в Outsee через API или CDP.
  2.2. Дождаться результата (таймаут: 120 сек по умолчанию).
  2.3. При успехе: скачать изображение.
  2.4. Сохранить в: data/series/{series_id}/images/{scene_id}_{shot_id}_{R4X}.jpg
       Пример: data/series/e01/images/scene_03_shot_01_R45.jpg
  2.5. Обновить ячейку R66 для этого кадра/ракурса:
       - Успех: «картинки_ок»
       - Ошибка: «переделать» (с записью причины в заметках, если есть поле)

ШАГ 3. После обработки всей очереди:
  3.1. Сохранить обновлённый Excel.
  3.2. Отправить уведомление: «Галерея готова: N картинок, M ошибок».

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
§5. МЯГКИЙ ПОВТОР (SOFT RETRY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

При ошибке генерации (CDP timeout, API error, пустой ответ):

  ПОПЫТКА 1: Ждать 5 сек, повторить запрос.
  ПОПЫТКА 2: Ждать 15 сек, повторить запрос.
  ПОПЫТКА 3: Ждать 30 сек, повторить запрос.
  ПОПЫТКА 4: Пометить «переделать» в R66, перейти к следующему промту.

ВАЖНО: При CDP timeout (потеря соединения с браузером):
  → НЕ удалять уже сгенерированные файлы других кадров серии.
  → НЕ сбрасывать R66 = «картинки_ок» для ранее успешных кадров.
  → Только помечать текущий сбойный кадр «переделать».
  → Пересоединиться с CDP и продолжить очередь с места сбоя.

Если пайплайн был прерван:
  → При перезапуске: пропускать ячейки с R66 = «картинки_ок».
  → Генерировать только «переделать» и пустые.

§6. ИМЕНОВАНИЕ ФАЙЛОВ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Шаблон: {series_id}/{scene_id}_{shot_id}_{R_row}.{ext}

Примеры:
  e01/scene_03_shot_01_R45.jpg   ← кадр1 сцены 3
  e01/scene_03_shot_02_R46.jpg   ← кадр2 (через плечо)
  e01/scene_03_shot_03_R47.jpg   ← кадр3 (реакция)
  e01/scene_07_shot_01_R45.jpg   ← кадр1 сцены 7

Базовая папка: data/series/ (конфигурируется в настройках пайплайна)
Формат: JPEG, качество 90%, 9:16 (вертикальный)

§7. ПАРАЛЛЕЛЬНОСТЬ И КВОТА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Outsee API квота:
  - По умолчанию: 1 запрос одновременно (sequential).
  - Если Outsee поддерживает batch: отправлять пачками по 5.
  - Задержка между запросами: 2–5 сек (чтобы не получить rate-limit).

Для серии из 50 ракурсов × 3 кадра = 150 изображений:
  При sequential с задержкой 3 сек: ~7.5 минут (плюс время генерации).
  Сообщить пользователю о прогрессе каждые 10 картинок.

§8. СТАТУС R66 — ДЕТАЛЬНОЕ ОПИСАНИЕ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

R66 — строка на листе «план», общая для всего столбца (не только для одного блока кадра).
Пайплайн должен обновлять R66 для каждого блока кадра независимо.

Возможные значения:
  (пусто)           : промта нет или не обрабатывался
  «ждёт_генерации»  : промт есть, генерация ещё не запускалась
  «в_очереди»       : поставлен в очередь Outsee
  «генерируется»    : Outsee обрабатывает
  «картинки_ок»     : файл получен и сохранён ← основной успешный статус
  «переделать»      : ошибка или отклонено человеком

Для ser_13_anim_prompts: он ищет «картинки_ок» и только тогда пишет R48/R64/R65.

Если R66 = «картинки_ок» и R67 = «да» (после picture lock) — НЕЛЬЗЯ
перегенерировать этот кадр без явной команды оператора.

§9. ВЗАИМОДЕЙСТВИЕ С OUTSEE (CDP)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Outsee работает через CDP (Chrome DevTools Protocol) на localhost:29229.
Это нужно только для реального запуска пайплайна, не для тестов.

Шаги работы с Outsee:
  1. Подключиться к Chrome via CDP (localhost:29229)
  2. Навигировать на страницу генерации outsee.io
  3. Вставить промт в поле ввода
  4. Выбрать параметры: aspect ratio 9:16, стиль проекта (если настраивается)
  5. Запустить генерацию
  6. Ждать появления результата (polling каждые 3 сек)
  7. Скачать изображение через click / download

При ошибке CDP:
  - Не завершать весь batch
  - Переподключиться (reconnect) и продолжить
  - Максимум 3 попытки переподключения подряд
  - После 3 неудач: записать в лог, пометить текущий кадр «переделать», перейти к следующему

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

§11. ЧЕЛОВЕЧЕСКИЙ СТОП-КРАН: ПРОСМОТР ГАЛЕРЕИ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

После завершения генерации:
  1. Studio UI отображает галерею: все «картинки_ок» кадры по сценам.
  2. Человек для каждого кадра: «принять» (R66 остаётся) или «переделать» (R66 = «переделать»).
  3. Для «переделать»: человек (или ser_12 повторно) пишет новый промт в R45/R46/R47.
  4. Пайплайн перегенерирует только «переделать» ячейки.
  5. Повторный просмотр для переделанных.
  6. После одобрения всех используемых ракурсов: нажать «Запустить ser_13».

Пайплайн НЕ запускает ser_13 автоматически без явного сигнала человека.

§12. ДИАГНОСТИКА ЧАСТЫХ ПРОБЛЕМ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ПРОБЛЕМА: Все картинки «переделать» сразу
  Причина: CDP не подключён / Outsee недоступен.
  Решение: Проверить Chrome на localhost:29229. Перезапустить Chrome.
    Команда: Start-Process chrome «--remote-debugging-port=29229» (Windows)

ПРОБЛЕМА: Картинки генерируются, но не того стиля
  Причина: Промт не содержит стиль проекта.
  Решение: Проверить R45: есть ли в конце «cinematic realism, 9:16 vertical, no text»?
    ser_12 должен добавлять стиль в конце каждого промта.

ПРОБЛЕМА: Одни и те же картинки для разных ракурсов
  Причина: Outsee кешировал промт.
  Решение: Добавить небольшое различие (seed, температуру) при повторной генерации.

ПРОБЛЕМА: R66 не обновляется
  Причина: Excel заблокирован другим процессом.
  Решение: Закрыть Excel, перезапустить пайплайн.

ПРОБЛЕМА: Файл скачан, но пустой (0 байт)
  Причина: Outsee вернул ошибку или таймаут.
  Решение: Удалить файл, пометить «переделать», повторить.

§13. API ИНТЕГРАЦИЯ (АЛЬТЕРНАТИВА CDP)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Если Outsee предоставляет REST API (или используется другой движок):
  POST /api/generate/image
    {
      "prompt": "<текст из R45>",
      "aspect_ratio": "9:16",
      "style": "<стиль проекта>",
      "output_format": "jpeg"
    }
  Ответ: { "image_url": "...", "status": "complete" }

Параметры для API:
  aspect_ratio : «9:16» всегда для вертикального видео
  quality      : «high» (или 1024x1792 px)
  style_preset : из настроек проекта (например «cinematic» или «painterly»)

Логика retry при API-ошибке: та же, что при CDP (§5).
