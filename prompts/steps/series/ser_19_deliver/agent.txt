§4. ЧТО ПИШЕШЬ (КОНТРАКТ ВЫВОДА)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4.1. ЛИСТ «СЕЗОН»

  В карте серий для текущего эпизода:
    статус : «выпущена»
    дата_выпуска : [дата если известна; иначе «выпущена»]
    финальный_файл : путь к файлу (из Монтажа или пайплайна)
    крючок_след : обновить крючок для следующей серии (если изменился по итогу)

  НЕ трогай другие серии в карте, кроме текущей.

4.2. ЛИСТ «НЕПРЕРЫВНОСТЬ»

  Добавляй строки только для фактов, которые:
  a) Стали НОВЫМИ в этой серии (в Непрерывности их ещё нет)
  b) ИЗМЕНИЛИ статус существующего факта (активно → закрыто; закрыто → активно)

  Каждая новая строка:
    факт         : что случилось / что теперь верно
    с_серии      : E01 (текущая)
    до_серии     : E-N если временное (закончится через N серий; иначе пусто)
    тип          : сюжет / одежда / рана / знание / место / отношения
    статус       : активно

  Для изменившихся статусов:
    Если факт закрылся в этой серии (рана зажила, секрет раскрыт):
      статус → закрыто; закрыто_в = EXX

  НЕ добавляй факты, которые уже были в Непрерывности до этой серии
  и не изменились.
  НЕ добавляй предположения и интерпретации — только конкретные факты.

4.3. КРАТКИЙ ОТЧЁТ ВЫПУСКА

  Текстовый блок в ответе агента (не в Excel):
    Серия: E0N «Название»
    Статус: выпущена
    Финальный файл: [путь или «звук TBD»]
    Длительность: [из Монтажа]
    Новые факты добавлены в Непрерывность: N
    Изменены факты в Непрерывность: M
    Крючок для E(N+1): [краткое описание]
    Предупреждения: [если есть]

§5. ПРАВИЛА ЗАПИСИ В «НЕПРЕРЫВНОСТЬ»
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

5.1. ТИПЫ ФАКТОВ

  «сюжет»: события, которые произошли и не откатятся.
    Пример: «Маша узнала, что Петя лжёт об алиби».

  «одежда»: изменение постоянного внешнего вида персонажа.
    Пример: «Петя потерял пальто в S07; далее без пальто».
    Временные изменения (промокший свитер на одну сцену) — НЕ вносить.

  «рана»: травмы с последствиями.
    Пример: «c01 Маша: порез на левой руке (S05)».
    Статус: активно → закрыто когда зажило.

  «знание»: что персонаж теперь знает (или узнал).
    Пример: «c02 Петя знает, что Маша обнаружила улику».

  «место»: изменение местонахождения персонажа на постоянной основе.
    Пример: «c03 Николай уехал из города (финал E01)».

  «отношения»: изменение отношений персонажей.
    Пример: «c01 и c02 в открытом конфликте после E01».

5.2. ТОЧНОСТЬ И ИСТОЧНИК

  Каждый факт должен быть верифицирован по «Сценарий» или «план» этой серии.
  Не вноси предположения («возможно, Маша догадывается»).
  Только то, что однозначно произошло на экране.

5.3. ОБЪЁМ

  Типичное количество новых фактов за серию: 3–10.
  Если получается больше 15 — перепроверь: не дублируешь ли то, что уже есть?
  Если меньше 2 — убедись, что это не неполный разбор.

6.1. СТАТУС СЕРИИ

  В карте сезона — поле статуса текущей серии меняется с «в_работе» / «монтаж»
  на «выпущена». Никаких других статусов не допускается.

6.2. КРЮЧОК ДЛЯ СЛЕДУЮЩЕЙ СЕРИИ

  Крючок (hook) — то, что цепляет зрителя смотреть дальше.
  Он должен быть:
  - Основан на событии, которое реально произошло в этой серии
  - Конкретным («Маша находит конверт с адресом» — не «что-то важное случилось»)
  - Записан в поле «крючок_след» текущей серии в «Сезон»

  Если крючок уже был записан библией сезона (ser_01_bible) — проверь,
  соответствует ли он тому, что реально произошло. Обновляй при расхождении.

6.3. АРКИ ГЕРОЕВ

  Если в «Сезон» есть раздел «арки героев» — обновляй прогресс по аркам
  для каждого главного персонажа, появившегося в серии.
  Не описывай целую арку — только сдвиг за эту серию.
