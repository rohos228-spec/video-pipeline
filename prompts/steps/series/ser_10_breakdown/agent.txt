## ПРАВИЛА ДЛЯ КАЖДОГО КЛЮЧЕВОГО ПОЛЯ
════════════════════════════════════════════════════════════════════════

R2 — id scene:
  — Формат: scene_XX (двузначный номер, начиная с 01 в каждой серии)
  — Уникален в рамках серии, не сквозной по сезону
  — Пример: scene_01, scene_02 … scene_17
  — Если сцена — флэшбек: scene_01_fb (добавить суффикс _fb)

R7 — фон (id):
  — Только id из листа «Фоны» (f01, f02_night, …)
  — Если место «день» → использовать f0X или f0X_day
  — Если место «ночь» → использовать f0X_night
  — Если в Фонах нет нужного id → СТОП: записать в конце отчёта «Требуется создать фон: [название]»
  — Не придумывать места, не входящие в «Фоны»

R8 — персонажи:
  — Только id из листа «Персонажи» (c01, c02 …)
  — Включить ВСЕХ участников сцены — и тех, кто говорит, и тех, кто молча присутствует
  — Массовка: если важна как группа → «массовка» или специальный id (cm01 …)
  — Порядок: главный персонаж первым, второстепенные после

R9 — предметы:
  — Только id из листа «Предметы» (p01, p02 …)
  — Включать только предметы, **активные в данной сцене** (присутствующие физически)
  — Проверить R7–R8 в карточке предмета: у_кого = один из R8 этого столбца?
  — Если предмет не активен в сцене — не включать

R13 — заметки непрерывности:
  — Кратко: «c02: хромает (F008), серая куртка (F012)»
  — Только активные факты, влияющие на ВИЗУАЛЬНОЕ ОТОБРАЖЕНИЕ в кадре
  — Нет нужды копировать всё из Непрерывности — только то, что важно для генерации картинки

R51 — тип покрытия:
  — СТРОГО одно из: диалог | давление | ссора | идут | толпа | нарезка | экшен | монолог | одиночная_сцена
  — Это предварительный тип — ser_11 может уточнить
  — Влияет на рецепт ракурсов в ser_11

R52 — главное действие:
  — Не более 8 слов: «допрос, Артём давит на Машу»
  — Не перефразировать весь сюжет — только ядро действия

R53 — день истории:
  — Сквозной счётчик по всему сезону (не сбрасывается при смене серии)
  — Если флэшбек — указать реальный день: «День -5 (флэшбек)»
  — Если тот же день, что и предыдущая сцена → тот же номер
  — Смена дня — только если явно показана в сценарии (новая утренняя сцена, сон/пробуждение)

R59 — номер кластера:
  — Может быть: «Акт 1», «Акт 2», «Акт 3» в рамках серии — если структура серии трёхактная
  — Или просто: «E02» — номер серии
  — Нужен для группировки при монтаже

  Для каждой сцены S01, S02 … создать столбец на листе «план». Порядок столбцов = порядок сцен в «Сценарии».

ШАГ 5 — ЗАПОЛНИТЬ ПОЛЯ КАЖДОГО СТОЛБЦА

    a) R2: scene_XX (нарастающий)
    b) R3: shot_01 (черновик)
    c) R4: scene_XX_shot_01

    h) R13: активные факты непрерывности по персонажам этой сцены
    i) R14: «разбор»
    j) R49: пусто (или текст рассказчика если есть, НЕ диалог)

    m) R53: день истории (нарастающий)
    n) R54: название места (текстом)
    o) R55: акцент камеры (черновик)
    p) R56: смысл/цель сцены из «Сценария»
    q) R57: тип сцены
    r) R58: особенности (флэшбек, спецэффект, монолог и т.д.)

ШАГ 6 — ПРОВЕРКА ССЫЛОК

  Ночная сцена на кухне = f03_night, а не f03. Промт фона кардинально отличается.

ОШИБКА 5 — Включить в R9 предмет, которого нет в сцене
  R9 — только предметы, физически присутствующие в данной сцене по сценарию.

ОШИБКА 6 — Не указать все персонажи в R8
  Если в сцене трое — все три id. Не забывать молчащих присутствующих.

ОШИБКА 7 — Установить R14 не «разбор», а что-то другое
  R14 при выходе этого агента = всегда «разбор». ser_11 потом изменит на следующий статус.

ОШИБКА 8 — Не сохранить сквозной счётчик дня истории между сериями
  День 14 в конце E02 → в E03 сцены той же ночи = День 14, сцены следующего утра = День 15.
