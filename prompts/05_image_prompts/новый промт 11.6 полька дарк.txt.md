ЧАСТЬ 1. ТЕХНИЧЕСКАЯ ЧАСТЬ
==========================

## 1.1. Назначение шаблона

Шаблон фиксирует:

* визуальный стиль;
* композиционную дисциплину;
* правила генерации image prompt;
* правила неповтора;
* правила физического текста;
* правила работы с исходными данными;
* правила prompt для кадра 1, кадра 2 и кадра 3.

Шаблон не фиксирует:

* коридор;
* силуэт;
* умывальник;
* таз;
* стену расследования;
* культовую тему;
* конкретную сцену;
* повторяющийся реквизит;
* доску расследования как обязательный визуальный приём.

Все сюжетные элементы, предметы, персонажи, локации и действия — переменные.

Их можно использовать только если они явно есть в исходных данных кадра, строке таблицы, описании персонажей, описании предметов, времени, месте, эпохе или конфликте сцены.

---

## 1.2. Обязательное указание ID персонажа

Если в кадре есть персонаж, prompt обязан явно указывать его ID.

Правильно:

char_01 stands near the window...

char_02 holds the medical folder...

char_03 turns away from the table...

Неправильно:

the man stands near the window...

the doctor holds the folder...

the woman looks afraid...

Если имя персонажа известно, ID всё равно обязателен.

Формат:

char_01 / Friedrich Nietzsche

char_02 / doctor

char_03 / nurse

Если ID персонажа не указан в исходных данных, нельзя выдумывать ID. Нужно написать:

нет исходных данных для ID персонажа

Если персонажа нет в кадре, ID не указывается.

---

## 1.3. Главное правило работы с исходными данными

Заполнять можно только те поля, для которых во входных данных есть явная информация.

Запрещено заменять отсутствующие данные догадками.

Запрещено придумывать:

* новых персонажей;
* новые предметы;
* новые документы;
* новые локации;
* новые символы;
* новые детективные элементы;
* новые надписи;
* новые сюжетные подсказки;
* новые prompt-блоки;
* дополнительные варианты кадров.

Если данных для поля нет, поле остаётся пустым или помечается:

нет исходных данных для заполнения

---

## 1.4. Лимит длины готового prompt

Каждый готовый prompt для одного кадра должен быть не длиннее 4900 символов вместе с пробелами.

Рабочий безопасный лимит:

до 4877 символов вместе с пробелами.

Если prompt получается длиннее 4877 символов, его обязательно сжать до лимита до финального вывода.

При сокращении удалять:

* повторы;
* воду;
* повторные перечисления стиля;
* дубли negative prompt;
* одинаковые запреты;
* лишние пояснения.

Нельзя удалять суть кадра:

* ID персонажа;
* главный субъект;
* место;
* время;
* действие;
* фокус;
* план;
* ракурс;
* свет;
* палитру;
* стиль;
* историческую достоверность;
* контекстную логику;
* запрет копирования строки 49;
* запрет voiceover context;
* правило физического текста;
* правило заранее определённых надписей;
* запрет нефизического текста на экране;
* запрет повтора текста на изображении у ближайших 10 сцен;
* запрет синонимичных сцен;
* запрет вымышленных деталей;
* правило неповтора;
* правило ограничения detective board: не чаще 1 раза на 15 prompts.

Запрещено отдавать prompt длиннее 4900 символов при любых условиях.

Перед финальным выводом обязательно проверить длину каждого prompt вместе с пробелами.

---

## 1.5. Жёсткое правило против копирования строки 49 и voiceover

Строка 49 и voiceover используются только как скрытый источник смысла.

Запрещено полностью или частично переписывать в prompt:

* строку 49;
* voiceover;
* закадровый текст;
* фразы из строки 49;
* отдельные характерные формулировки из строки 49;
* метафоры из строки 49;
* объяснительные конструкции из строки 49;
* рассуждающий текст из voiceover.

В prompt запрещены блоки и формулировки:

* Voiceover context:
* voiceover context:
* based on the voiceover phrase
* the narration says
* the text explains
* the scene symbolizes the phrase
* visual metaphor for the voiceover
* the image shows the idea that

Нельзя писать prompt как пересказ закадрового текста.

Нужно превращать смысл строки 49 в физически видимое действие, состояние, предмет, позу, свет, пространство или результат.

Если текст не является предметным и не содержит конкретного объекта, допускается хоррор-абстракция на основе данных столбца, но она должна быть визуальной, а не текстовой.

Допустимая хоррор-абстракция:

* пустой коридор с искажённой тенью;
* фигура у двери в неестественно низком ракурсе;
* холодный свет на пустой кровати;
* тёмный силуэт в глубине комнаты;
* предмет, оставленный в пустом пространстве;
* архитектура, давящая на персонажа;
* красные grunge-следы как абстрактное напряжение стиля, не как текст и не как подсказка.

Запрещённая хоррор-абстракция:

* надпись поверх кадра;
* рассуждающая фраза на изображении;
* текстовая метафора;
* случайная детективная карта;
* карта причин;
* доска расследования без прямого основания;
* красные стрелки, круги и подписи вокруг объектов.

Для PROMPT_1 можно использовать данные только из КАДРА 1.

Нельзя переносить в PROMPT_1 данные из КАДРА 2 или КАДРА 3.

## 1.7. Новое жёсткое правило для кадра 2

PROMPT_2 больше нельзя копировать из строк 26 + 28.

PROMPT_2 создаётся как акцентный prompt на основе референса.

PROMPT_2 обязан найти в исходных данных кадра 2 один главный элемент:

1. главное действие;
2. или главный предмет;
3. или главную эмоцию.

Затем этот элемент нужно показать:

* крупным планом;
* или выделенным визуальным акцентом;
* с обязательной сменой положения камеры;
* с экстремальным ракурсом.

PROMPT_2 обязан начинаться точной фразой:

«на основе референса, запрещено делать идентичную иллюстрацию без смены положения камеры»

После этой фразы нужно кратко написать:

1. что должно быть в кадре;
2. план;
3. действие;
4. экстремальный ракурс.

Запрещено:

* копировать строку 26;
* копировать строку 28;
* копировать строку 49;
* копировать voiceover;
* делать идентичную иллюстрацию;
* повторять PROMPT_1;
* сохранять тот же ракурс камеры;
* сохранять тот же план;
* писать новый полный cinematic prompt;
* добавлять новые предметы;
* добавлять новых персонажей;
* добавлять новые надписи;
* добавлять detective board без прямого основания.

Формат PROMPT_2:

на основе референса, запрещено делать идентичную иллюстрацию без смены положения камеры; в кадре [главное действие / главный предмет / главная эмоция]; [крупный план / экстремально крупный план / деталь / низкий ракурс / вид сверху / резкий диагональный угол]; [что происходит физически]; [какой экстремальный ракурс меняет композицию]; сохранить стиль, эпоху, персонажа с ID, световую и цветовую логику референса, но изменить положение камеры.

Примеры допустимых экстремальных ракурсов:

* extreme low angle;
* extreme high angle;
* overhead view;
* Dutch angle;
* close-up from behind the object;
* distorted perspective through doorway;
* profile close-up with compressed background;
* top-down view on hands;
* worm’s-eye view;
* sharp diagonal angle;
* over-the-shoulder extreme close-up;
* camera almost at floor level;
* camera pressed close to the object;
* fragmented close-up through shadow.

Если во втором кадре есть персонаж, его ID обязателен.

Если второго кадра нет, PROMPT_2 не создаётся.

---

## 1.8. Жёсткое правило для кадра 3

PROMPT_3 создаётся только если во входных данных явно есть КАДР 3.

Если КАДР 3 не указан, не описан или отсутствует, PROMPT_3 запрещён.

Нельзя создавать третий кадр:

* автоматически;
* для полноты;
* как продолжение сцены;
* по аналогии;
* ради визуального разнообразия.

Если КАДРА 3 нет, нужно указать:

КАДР 3: нет исходных данных для заполнения.

* red arrows pointing at evidence;
* literal investigation-board markings;

## 2.10. Composition vector

COMPOSITION_VECTOR =

one unified scene, not a collage; chaotic collage energy translated into integrated poster composition; single readable focal point; controlled cinematic framing; medium-distance composition by default; high visual tension; readable spatial depth without forcing large foreground objects; minimal unobtrusive foreground; no oversized props at the front edge; no repeated foreground-object formula; no detective board logic unless the source data explicitly requires a real physical board and detective-board frequency limit allows it; no synonym scene without direct need.

* силуэт на фоне стены;
* ощущение угрозы через масштаб, свет, пустоту и позу.

Запрещено:

* добавлять текстовую метафору;

ЧАСТЬ 3. ПРЕДМЕТНАЯ ЧАСТЬ: КАК КАДР СОБИРАЕТСЯ
==============================================

## 3.1. Главный принцип сборки кадра

Кадр строится не через случайный предмет-улику.

Кадр строится через:

* ID персонажа;
* действие персонажа;
* положение тела;
* эмоцию через жест, лицо или позу;
* среднюю дистанцию;
* свет;
* архитектуру;
* фактуру;
* контекст;
* историческую среду;
* последствия действия;
* расстояние между объектами;
* направление взгляда;
* физически поддержанные детали.

Передний план не является обязательным местом для важного предмета.

Нельзя автоматически добавлять крупные foreground props:

* тазы;
* миски;
* чаши;
* лампы;
* столы;
* документы;
* оружие;
* книги;
* карты;
* улики;
* мебель;
* случайный реквизит у переднего края кадра.

---

## 3.2. Правило контроля переднего плана

FOREGROUND_CONTROL_RULE =

Keep the main subject in the midground or readable central area by default.

Не заставлять модель помещать важный предмет у переднего края изображения.

По умолчанию главный субъект должен находиться:

* в читаемой центральной зоне;
* или в среднем плане.

Передний план должен быть:

* минимальным;
* спокойным;
* не доминирующим над сценой.

Запрещено:

* автоматически добавлять крупные предметы у нижнего края кадра;
* использовать foreground object как стандартную замену контекстной логики;
* использовать формулу «крупная вещь на переднем плане + персонаж вдали», если это прямо не требуется исходным кадром;
* автоматически использовать крупные документы, миски, чаши, тазы, лампы, столы, книги, ножи, карты, улики, перчатки, ключи или мебель как foreground anchor.

Глубина сцены должна строиться через:

* архитектуру;
* свет;
* позу;
* расстояние;
* фактуру стен;
* фактуру пола;
* двери;
* окна;
* мебель;
* окружение.

Если исходные данные не требуют предмет на переднем плане, не добавлять его.

---

## 3.3. Правило запрета медных тазов и повторяющихся сосудов

Запрещены, если прямо не указаны в исходных данных:

* copper basin;
* brass basin;
* metal basin;
* washbasin;
* washstand;
* metal bowl;
* old bowl;
* water bowl;
* washing bowl;
* таз;
* медный таз;
* латунный таз;
* металлическая миска;
* умывальный таз;
* умывальник;
* любые похожие сосуды.

Нельзя использовать таз, чашу, миску, умывальник или сосуд как универсальный исторический реквизит.

Если в исходном кадре есть старое помещение, больница, комната, архив, монастырь, казарма или кабинет, это не является основанием добавлять таз, миску или умывальник.

Если сосуд прямо указан в исходных данных, он может появиться:

* только один раз;
* только в логичном месте;
* без увеличения до доминирующего foreground-объекта.

---

## 3.4. Контекстные детали и запрет вымысла

CONTEXT_DETAIL_RULE =

Every important visible detail must come from the frame data, video context, location, time period or stated conflict.

На изображении должны быть визуальные детали, связанные с:

* данными кадра;
* данными столбца;
* локацией;
* временем;
* исторической эпохой;
* описанным конфликтом;
* персонажем;
* предметом, если он прямо указан.

Запрещено делать пустую атмосферную картинку без связи с контекстом.

Запрещено добавлять:

ЧАСТЬ 6. КОГДА ПОКАЗЫВАЮТСЯ ДЕТЕКТИВНЫЕ ДОСКИ
=============================================

## 6.1. Главное правило

Детективные карты, доски расследования, evidence boards, clue maps, стены расследования, карточки с нитями и схемы связей запрещены как частый визуальный приём.

Такие изображения допустимы не чаще:

1 раз на 15 prompts.

То есть:

* 1–15 prompts → максимум 1 detective board / clue map;
* 16–30 prompts → максимум 2;
* 31–45 prompts → максимум 3;
* 46–60 prompts → максимум 4.

Если лимит уже использован, следующая detective board запрещена, даже если она визуально красивая.

---

## 6.2. Когда detective board разрешена

Detective board, investigation wall, clue map или карта связей разрешена только если одновременно выполнены все условия:

1. В исходных данных прямо есть детективная логика.
2. В сцене действительно много переменных, которые нужно физически сопоставить.
3. Есть прямое основание для доски, карты, карточек, нитей или схемы.
4. Доска или карта является реальным физическим объектом внутри сцены.
5. Это один из самых явных детективных моментов в блоке.
6. Без доски зритель хуже поймёт связь множества элементов.
7. Лимит 1 раз на 15 prompts не превышен.
8. Кадр с доской не повторяет ближайшие кадры.
9. Нити, карточки и связи находятся только на самой доске или физической рабочей поверхности.
10. Нет графических линий, стрелок, кругов и подписей поверх изображения.

---

## 6.3. Когда detective board запрещена

Доска расследования запрещена, если:

* в исходных данных нет доски;
* в исходных данных нет карты;
* в исходных данных нет карточек;
* в исходных данных нет нитей;
* нет множества переменных;
* сцена не является явно детективной;
* это обычная эмоциональная сцена;
* это обычная историческая сцена;
* это бытовая сцена;
* это больничная сцена без расследовательского действия;
* это архивная сцена без явной схемы связей;
* доска нужна только как красивая визуализация;
* доска заменяет мысль из строки 49;
* доска копирует voiceover;
* доска появляется чаще 1 раза на 15 prompts;
* доска похожа на предыдущую detective board;
* доска создаёт синонимичную сцену.

---

## 6.4. Что запрещено даже при разрешённой доске

Запрещены:

* красные линии поверх персонажей;
* красные линии поверх лиц;
* красные линии поверх улиц;
* красные линии поверх комнат;
* красные линии поверх документов;
* красные линии поверх фона;
* стрелки поверх изображения;
* круги вокруг улик;
* target marks;
* evidence circles;
* red circles;
* red outlines around clues;
* red arrows pointing at evidence;
* literal investigation-board markings;
* графические связи поверх всей сцены;
* рассуждающий текст на доске;
* текст из строки 49 на доске;
* voiceover phrase на доске;
* философские подписи на доске;
* неподготовленные надписи;
* придуманные названия дел;
* случайные даты;
* случайные имена.

Линии, нити, карточки и связи могут быть только:

* на самой физической доске;
* или на рабочей поверхности, если она прямо указана в исходных данных.

## 6.5. Как передавать расследовательскую логику без доски

В обычных кадрах расследовательская логика передаётся через:

* позу;
* свет;
* предметы сцены;
* следы;
* дистанцию между объектами;
* направление взгляда;
* композицию;
* причинно-следственные детали;
* фактуру пространства;
* последствия действия;
* пустое место;
* изношенные поверхности;
* физические следы;
* отношение персонажа к среде;
* заранее определённые надписи на реальных предметах, если они заданы.

Если в исходных данных нет:

* доски;
* стены;
* карточек;
* нитей;
* прямого указания на investigation board,

их нельзя добавлять.

## 6.6. Board usage внутри prompt

BOARD_USAGE =

[указать, есть ли доска, почему она допустима, является ли она физическим объектом, не превышен ли лимит 1 раз на 15 prompts, не превращается ли сцена в investigation wall без основания]

Если доски нет, писать:

BOARD_USAGE =

no detective board, no investigation wall, no strings, no clue map; context is shown through physical environment, posture, light, source-supported objects and spatial tension.

Если доска есть и она прямо поддержана исходными данными, писать:

BOARD_USAGE =

real physical investigation board inside the scene, explicitly required by source data; allowed by the 1-in-15 prompt limit; cards, notes and strings remain only on the board surface, not overlaid across the frame; no red circles, no target marks, no arrows, no graphic evidence overlays, no copied row 49, no voiceover text.

## 6.7. Board frequency check

BOARD_FREQUENCY_CHECK_1_OF_15 =

[указать номер текущего prompt в блоке и использовалась ли detective board в предыдущих 15 prompts]

Если уже была detective board в ближайших 15 prompts, писать:

BOARD_FREQUENCY_CHECK_1_OF_15 =

detective board already used within nearest 15 prompts; board is forbidden in this prompt.
